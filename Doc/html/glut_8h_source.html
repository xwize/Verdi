00080 /* Win32 has an annoying issue where there are multiple C run-time
00081    libraries (CRTs).  If the executable is linked with a different CRT
00082    from the GLUT DLL, the GLUT DLL will not share the same CRT static
00083    data seen by the executable.  In particular, atexit callbacks registered
00084    in the executable will not be called if GLUT calls its (different)
00085    exit routine).  GLUT is typically built with the
00086    "/MD" option (the CRT with multithreading DLL support), but the Visual
00087    C++ linker default is "/ML" (the single threaded CRT).
00088 
00089    One workaround to this issue is requiring users to always link with
00090    the same CRT as GLUT is compiled with.  That requires users supply a
00091    non-standard option.  GLUT 3.7 has its own built-in workaround where
00092    the executable's "exit" function pointer is covertly passed to GLUT.
00093    GLUT then calls the executable's exit function pointer to ensure that
00094    any "atexit" calls registered by the application are called if GLUT
00095    needs to exit.
00096 
00097    Note that the __glut*WithExit routines should NEVER be called directly.
00098    To avoid the atexit workaround, #define GLUT_DISABLE_ATEXIT_HACK. */
